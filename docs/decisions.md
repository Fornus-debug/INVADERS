# Decisions

## 2026-06-22

- `src/` と `package.json` が存在しないため、正本は `katakata-invaders.html` と判断した。
- 画面遷移は hash route 風の page view 切り替えを採用した。
- CTAリンクは単体HTML内の `LINKS` 定数管理を維持した。
- LINE CTAは `https://lin.ee/RnwJwMO`、EメールCTAは `mailto:fornus.fujiwara@gmail.com` で有効化した。
- PC表示は1画面固定ではなく、内容が切れないことを優先して縦スクロール可能にした。

## 2026-06-23

- 正本は `katakata-invaders.html` と判断し、React / Vite構成の新規復元は行わない。
- `#/chatbot` に業務改善相談AIを追加した。
- 最初のMVPではベクトルDBを使わない。
- ナレッジは `docs/knowledge/*.md` で管理する。
- 外部AI APIは現時点では呼ばない。APIキー未設定でも壊れないことを優先し、検索結果から簡易回答を生成する。
- 回答形式は「共感」「問題整理」「改善案」「おすすめツール」「類似事例」に固定する。
- 新規ライブラリは追加しない。

## 2026-06-24

- チャット画面を最初から表示せず、カタカタくんの起動画面を先に表示する構成にした。
- 通常ボタンではなく、キャラクターのお腹の `ENTER` ボタンを押して相談開始する導線にした。
- 吹き出しメッセージはループ切り替えで実装した。
- Enterボタンはホバー / フォーカスで発光と拡大、クリックで沈む演出を付けた。
- 既存RAG検索、回答生成、類似事例表示の処理は維持した。
- スマホではキャラと吹き出しが切れないよう、720px以下と480px以下でサイズを調整した。

## 2026-06-27

- `#/chatbot` を「カタカタ改善診断AI」として再設計した。
- 既存の単体HTML構成、レトロゲームUI、カタカタくん起動画面は維持した。
- まずは静的HTML内のルールベース診断として実装し、外部AI APIは接続しない。
- ファイルアップロードはMVPではファイル名表示と相談文反映のみとし、実ファイル解析はバックエンド導入後に行う。
- 診断人格は「AI改善係長」に統一する。
- 診断結果は、AI改善係長コメント、ムダ作業、属人化リスク、自動化候補、削減時間、実装難易度、見積もり目安、推奨ツール、最初にやるべき改善、類似事例、この内容で相談する、相談文コピーを表示する。
- 既存ポートフォリオUIを壊さず、WORKSとSERVICESへカタカタ改善診断AIを追加する方針にした。
- 営業デモで一発で見せられるよう、入力例、結果カード、コピーCTA、LINE相談CTA、再診断CTAを強化した。
- ファイルの中身解析は未実装であることを画面内にも明記し、「解析済み」に見せない。
- 公開デモ仕上げとして、トップ、メニュー、WORKS、SERVICES、CONTACT から `#/chatbot` へ誘導するCTAを追加した。
- OGP / meta は「カタカタ改善診断AI」とAI業務改善デモが伝わる内容へ更新した。
- `README.md`、`docs/demo-checklist.md`、`docs/demo-script.md` を追加し、公開前確認と営業説明に使う。
- 次は実機ブラウザ確認または公開先への配置を行う。
- GitHub Pagesなどの静的ホスティングでトップURLから開けるよう、`index.html` を `katakata-invaders.html` と同一内容で追加する。
- 既存の `katakata-invaders.html` は残し、`/#/chatbot` と `/katakata-invaders.html#/chatbot` の両方を公開確認対象にする。

## 継続方針

- 技術デモではなく、実際に相談できる営業用デモとして改善する。
- 「カタカタくんに相談する」体験を優先する。
- 製造業での改善経験をナレッジの強みにする。
